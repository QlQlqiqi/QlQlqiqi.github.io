Nginx 的负载均衡实验

一、nginx 用于代理或反向代理。

正向代理：连接在企业内网的主机，向外请求的时候，都会通过一个代理，用其的公网 IP 向外请求。

反向代理：处在公网中的主机向连接企业内网的主机请求，数据会通过一个公网 IP 先流向一个代理，然后通过它转发数据到该内网主机。

反向代理：

规定：localhost:80 是代理，localhost:8001、localhost:8002、localhost:8003、localhost:8004 分别是内网主机

二、均衡负载

在反向代理中，代理将请求发送到其他的服务器，但是将请求发送到哪个服务器所用的策略，都是为了更好地利用服务器资源对请求进行快速响应。

  1. 轮询

这是默认的方法，依次请求每一个服务器

xxxxxxxxxx
6
1
upstream myserver {
2
        server 127.0.0.1:8001;
3
        server 127.0.0.1:8002;
4
        server 127.0.0.1:8003;
5
        server 127.0.0.1:8004;
6
}

上面设置 weight 均为 1 时就是轮询的方法。

  1. 权重 weight

有的服务器性能好，所以可以给它分配更多的请求，weight 越大，给它分配的可能性越大

x
1
upstream myserver {
2
        server 127.0.0.1:8001 weight=1;
3
        server 127.0.0.1:8002 weight=2;
4
        server 127.0.0.1:8003 weight=3;
5
        server 127.0.0.1:8004 weight=4;
6
}

可以看到 8004 端口比 8001 更容易被分配请求。

  1. ip_hash

当用户登录了某个网站，如果再没有退出前又重新进入，如果这个请求被分配给了其他服务器，那么其登录信息会丢失，所以需要将其分配给之前的哪个服务器。

x
1
upstream myserver {
2
        ip_hash;
3
        server 127.0.0.1:8001;
4
        server 127.0.0.1:8002;
5
        server 127.0.0.1:8003;
6
        server 127.0.0.1:8004;
7
}

可以看到物理机和虚拟机分别请求特定的端口。

  1. fair（第三方）

有的服务器当前任务量大，给它少分配请求，所以可以根据服务器响应时间决定给谁分配。

xxxxxxxxxx

1
upstream myserver {
2
    server server1;

    server server2;
4
    fair;
5
}
  1. url_hash（第三方）

按照访问的 url 来分配服务器，可以更好的利用缓存。

 

tips：

  1. 如果你 curl proxy 有时候会出现 unkown request ，有可能该端口被占用

可以通过 netstat -aon|findstr "pid" 查看是否被占用

可以看到我的 8082 端口确实被占用了，那么就更换一个端口测试。

  1. 有时候更改好了nginx.conf 文件并 nginx -s reload ，但是发现代理不起作用，即使 start nginx 也不行，然后nginx -s stop 后发现原代理还在运行，看后台发现有很多 nginx.exe 在运行，是因为 start nginx 会重新启动 nginx ，可以通过 taskkill /f /IM nginx.exe 来结束所有的 nginx.exe ，并后续可以通过 nginx -s quit 来退出 nginx。
